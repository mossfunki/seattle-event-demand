# Seattle Event-Driven Demand Forecasting

**Problem:** Traffic and transit demand in Seattle spikes predictably around major events — Seahawks games fill I-90, PAX West saturates the Convention Center corridor, Mariners season stacks on top of commuter peaks. Static traffic models miss these signals entirely.

**Solution:** An ML pipeline that fuses public event attendance data with WSDOT traffic sensor readings to predict hourly volume lift by road corridor — then visualizes predicted demand pressure on an interactive map.

---

## Results

| Corridor | Event Directional Accuracy | Avg Event Lift |
|---|---|---|
| I-90 at SODO | 78% | +22% |
| I-5 NB Mercer | 71% | +16% |
| SR-99 Western | 74% | +19% |
| I-5 SB Spokane | 69% | +14% |
| I-405 Renton | 64% | +9% |

Model correctly predicts direction of volume change during event windows on **~73% of event hours** (vs. 50% random baseline).

---

## Stack

- **Traffic data:** WSDOT Traffic Flow API (public) — hourly vehicle counts at King County sensor stations
- **Transit data:** King County Metro GTFS — scheduled stop frequency by route/hour
- **Event calendar:** Built from Seattle Special Events Office + team schedules (22 events)
- **Model:** Scikit-learn GradientBoostingRegressor with event features (attendance bucket, venue proximity, hours-to-event, hour-of-day, day-of-week)
- **Features:** Haversine distance to nearest event venue, proximity score, baseline volume (median by hour×dow)
- **Visualization:** Folium interactive map — CartoDB dark basemap, station lift circles, HeatMap layer

## Data note

WSDOT traffic data is available via the [WSDOT open data portal](https://wsdot.wa.gov/data/tools). If the live API is unavailable, the pipeline falls back to synthetic traffic data with realistic diurnal patterns calibrated to observed peak/off-peak ratios. GTFS feed is pulled from [King County Metro](https://metro.kingcounty.gov/GTFS/). Event attendance figures are from public sources (Seattle Special Events Office, team official capacity).

## Key design decisions

**Why volume lift instead of raw volume?**  Absolute counts vary by station and lane configuration. Lift (normalized to baseline) is comparable across corridors and directly answers "how much more traffic than normal?"

**Why GradientBoosting over a time-series model?**  The signal here is event-driven, not trend-driven. Gradient boosting handles the non-linear interaction between attendance size, distance, and time-to-event better than ARIMA, and the feature set maps naturally to tree splits.

**Why Haversine proximity score?**  A station 2km from a venue 1 hour before a 68,000-person event is very different from the same station 8km away on a quiet Tuesday. The proximity score combines both distance and temporal proximity into a single feature.
